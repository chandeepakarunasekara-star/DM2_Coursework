Project Design

Relational Entity-Relationship (EER) Model & MongoDB Schema

Entity Relationship Design

The Oracle database utilizes a third-normal-form (3NF) relational schema designed to guarantee transactional integrity and support atomic academic operations.

CATEGORIES & COURSES

One-to-many relationship: A category classifies multiple courses; a course belongs to exactly one category.

LECTURERS & COURSES

One-to-many relationship: An instructor can teach multiple courses; each course is taught by one lecturer.

STUDENTS, ENROLLMENTS & COURSES

Many-to-many relationship resolved through the associative entity ENROLLMENTS. Backed by unique index composite keys to prevent duplicate active student registrations.

ENROLLMENTS & PAYMENTS

One-to-many relationship: Tracks the enrollment fee. Simulates invoicing and records payments with bank transaction references.

ENROLLMENTS & CERTIFICATES

One-to-one relationship: A student earns exactly one unique digital certificate upon reaching 100% lesson progress, enforced by unique keys and PL/SQL database triggers.

💡 Coursework Highlights

Our schema enforces constraints such as checks on payment amounts (>= 0), course prices (>= 0), student statuses, and unique indexes to defend against corrupted data values.

Schema Dictionary (Oracle Relational Tables)

Table Name Primary Key Foreign Keys Core Constraints & Notes
CATEGORIES category_id None category_name is UNIQUE and NOT NULL.
LECTURERS lecturer_id None email is UNIQUE. status must be 'ACTIVE' or 'INACTIVE'.
STUDENTS student_id None email is UNIQUE. status check: ACTIVE, SUSPENDED, GRADUATED.
COURSES course_id category_id, lecturer_id published_status CHECK (DRAFT, PUBLISHED, ARCHIVED). price >= 0 check.
LESSONS lesson_id course_id lesson_order UNIQUE per course_id (forces ordered module structures).
ENROLLMENTS enrollment_id student_id, course_id UNIQUE(student_id, course_id) prevents double-registration.
PAYMENTS payment_id enrollment_id transaction_ref is UNIQUE. payment_status CHECK (PENDING, PAID, etc).
LESSON_PROGRESS progress_id enrollment_id, lesson_id UNIQUE(enrollment_id, lesson_id). quiz_score checked between 0 and 100.
CERTIFICATES certificate_id enrollment_id enrollment_id is UNIQUE. certificate_code is UNIQUE.

MongoDB Document Collections

Collection Name Linked Keys Dynamic Structure Pattern
courseResources courseId Stores varied academic files (lecture PDFs, slide links, mp4 videos, quiz structures) using flexible nested object arrays.
courseReviews courseId, studentId Captures reviews, numeric ratings (1 to 5), and open-ended feedback comments. Optimized with indexing for aggregation.
discussionThreads courseId, authorId Models forums and Q&A boards. Employs text indexes on titles and post bodies for full-text query searches.

MongoDB Aggregations & Analytical Queries

In our architecture, MongoDB works as the high-throughput, unstructured document database. The following MongoDB commands satisfy the course brief's query specifications:

Query 1: Retrieve All Feedback for a Given Course 
db.courseReviews.find({
  courseId: "C101"
})
Query 2: Aggregated Course Ratings (Top Rated) 
db.courseReviews.aggregate([
  { 
    $group: { 
      _id: "$courseId", 
      averageRating: { $avg: "$rating" }, 
      reviewCount: { $sum: 1 } 
    } 
  },
  { $sort: { averageRating: -1, reviewCount: -1 } }
])
Query 3: Full-text Search on Forums 
db.discussionThreads.find({
  $text: { $search: "trigger" }
})

Hybrid Analytics Innovation

A key innovation of CourseConnect is combining relational transactional metrics from Oracle and engagement data from MongoDB to compute hybrid intelligence reports.

Relational + NoSQL Integration

The web application reads enrollment statistics, course pricing, and payments status from Oracle, then fetches student reviews and dynamic files from MongoDB, binding them dynamically in the Course Details Drawer.

Business Analytics

Using these joint queries, the system dashboard can flag courses that have high transactional revenues but declining feedback scores, allowing coordinators to optimize student engagement plans.
